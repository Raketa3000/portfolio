# Marketing & Creative Community

Одностраничный сайт креативного сообщества с полноэкранной навигацией, анимациями, локализованными кейсами и адаптивной версткой.

## Технологии

- HTML, CSS и JavaScript без сборщика и внешнего JavaScript-фреймворка;
- JSON-файлы для локализации интерфейса и контента кейсов;
- локальные изображения и видео для кейсов;
- Google Fonts для шрифта Onest.

## Структура проекта

```text
landing-page-prod/
├── index.html                 # Основная страница
├── ru/index.html              # Маршрут русской версии
├── eng/index.html             # Маршрут английской версии
├── es/index.html              # Маршрут испанской версии
├── assets/
│   ├── cases/
│   │   ├── en/                # Английские версии кейсов
│   │   ├── es-la/             # Испанские версии кейсов
│   │   └── media/             # Медиафайлы кейсов
│   ├── css/styles.css
│   ├── i18n/                  # UI-тексты RU, EN и ES-LA
│   ├── images/                # Фото команды и favicon
│   └── js/script.js
├── scripts/
│   ├── smoke-assets.js        # Проверка HTML, favicon, JSON и путей к медиа
│   └── smoke-i18n.js          # Проверка критичных локализованных строк
└── README.md
```

## Локальный запуск

Сайт загружает JSON через `fetch()`, поэтому его нужно запускать через HTTP-сервер, а не открывать как локальный файл.

```bash
python3 -m http.server 4173
```

После запуска доступны адреса:

- `http://127.0.0.1:4173/` — автоматический выбор языка;
- `http://127.0.0.1:4173/ru/` — русский;
- `http://127.0.0.1:4173/eng/` — английский;
- `http://127.0.0.1:4173/es/` — испанский для Латинской Америки.

## Локализация

Интерфейс хранится в:

- `assets/i18n/ru.json`;
- `assets/i18n/en.json`;
- `assets/i18n/es-la.json`.

Приоритет выбора языка:

1. языковой сегмент URL;
2. устаревший параметр `?lang=` с последующей заменой URL;
3. значение `site-lang` в `localStorage`;
4. языки браузера из `navigator.languages`;
5. русский язык как fallback.

Переключатель языка сохраняет выбор в `localStorage` и заменяет адрес на `/ru/`, `/eng/` или `/es/`. Неполные переводы интерфейса автоматически дополняются значениями из `ru.json`.

## Кейсы

Русские версии кейсов находятся в `assets/cases/*.json`. Английские и испанские версии используют ту же структуру и находятся в `assets/cases/en/` и `assets/cases/es-la/`. Если локализованный файл или отдельное поле отсутствует, используется русская версия.

Список загружаемых кейсов задается в `caseFiles` внутри `assets/js/script.js`. Медиафайлы размещаются в `assets/cases/media/`.

Пример блока `media`:

```json
{
  "media": [
    {
      "type": "image",
      "src": "assets/cases/media/case-image.webp",
      "alt": "Описание изображения",
      "caption": "Кадр из проекта",
      "fit": "contain"
    },
    {
      "type": "video",
      "src": "assets/cases/media/case-video.webm",
      "poster": "assets/cases/media/case-poster.webp",
      "caption": "Ролик проекта",
      "fit": "cover"
    },
    {
      "type": "embed",
      "html": "<iframe src=\"https://player.vimeo.com/video/000000000\" allowfullscreen></iframe>",
      "caption": "Внешний плеер"
    }
  ]
}
```

Если `media` не указан, кейс отображается без медиагалереи.

## Проверки

Перед публикацией запустите:

```bash
node scripts/smoke-i18n.js
node scripts/smoke-assets.js
```

Первая команда проверяет критичные строки локализации, вторая — обязательные элементы страницы, favicon, корректность JSON и существование локальных медиафайлов.

## Публикация на GitHub Pages

Проект не требует сборки. Для публикации выберите ветку `main` и корневую папку репозитория в настройках GitHub Pages.
